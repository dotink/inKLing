Introduction

The inKLing stylesheet is broken into two parts. The first part is a combination reset, sane defaults, and opinionated base styling. This is the base which all inKLing sites derive from and handles default structural integrity. The second part is a theme.

Theme's use inKLing's suggested classe names to define overall look and feel.

Themes define coloring, padding/margin overrides for the structural elements, and general background colors. Themes should not be so specific as to suggest an entire site design, but should be specific enough that they could be used to create a nice looking site rather quickly.

Lists

Lists in inKLing are expertly designed to function appropriately in standard and not-so standard circumstances. For example:

  • Nested list items are ensured to have a 1em font-size to avoid scaling down with nesting.
  • List items containg other block level elements will adjust their margins accordingly.

    • Multiple paragraphs can easily be contained in a single list item. This allows you to use ordered lists for purposes like legal contracts or licensing terms with different sections that are ordered.

      Since the margin is only stripped on the last child, your paragraphs will look consistent with the rest of your document. At the same time, your list items will not appear segregated from one another.

  • Lists themselves have a margin that is consistent with paragraphs.

Grouping List Items

In addition to standard lists like that above, you can also group list items by adding the group class directly on a ul or an ol element.

  • Grouped Lists Don't Have Bullets

    As you can see, grouped lists are nice and clean and have their bullets stripped the moment they're grouped

  • Headings Have Different Margins

    Any headings which are in grouped list items will have their top margin removed. This keeps the grouping nice and tight with any preceding heading.

  • Nested Lists Do the Following
    • Keep suitable list style positioning
    • Maintain consistent margins
      • Which helps sub-sub-lists

Because groups automatically clear, you don't need to worry about your grouped list items containing content of different lengths. Things just keep on flowing!

Other Common Groupings

There are three common grouping styles in inKLing. These use pseudo-semantic names and are designed for larger structrual elements. Their names are designed to be partially semantic in that most of the cases they would be used, it would carry some semantic meaning to what you're displaying.

Featurette

A .featurette.group is two equal width children by default similar to what you might use to feature items in a top of page billboard

Brochure

A .brochure.group is three equal width children by default similar to a tri-fold brochure.

Catalogue

A .catalogue.group is four equal width children by default similar to what you would use if you were presenting a large number of small equally important items

Containers

Containers (divisional elements with a class of container) are designed to be segregated from their surrounding content. That is, they are considered to be elements which are "free standing" in some regards.

Container Title

Container Padding

A container which is not a group will have padding applied to it so that the elements it contains are separated from it's edges.

Not Too Special

There is no special treatment of the elements contained within with the exception of first titles in a group adjacent to a title. Everything else should operate as any other element would in a parent, unaware of the padding which restricts them.

Container Groups

It is also possible to add the container class to a group itself. Unlike a mere .container that is not a group, a .container.group will have it's padding passed on to sections.

Padding

You can see from this example that each section is padded individually.

Background Colors

In addition to padding, each child element is padded heavily on the bottom with a negative matching margin, filling normally empty bottom space with the its background color, giving the impression of equal height columns.

Messaging

This is a notice. A notice is designed to give you additional information, but differs from a helper in that it is more informational than helpful, for example, "Images must be exactly 300x210 pixels." It is acceptable to use a notice outside of a form, while a helper should only be used in a form to help the user complete their task.

This is a helper. A helper supplies specific coaching text for a user, such as, "Click and drag the items to order them how you'd like."

This is an alert. Alerts must provide actional items such as "continuing" or "canceling" an action and are designed to intervene when a user should be aware of something before continue an action or taking an alternative route. The green button below has a class of suggested.
